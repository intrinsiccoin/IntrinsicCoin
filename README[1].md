<h2><a id="user-content-about" class="anchor" aria-hidden="true" href="#about"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>About</h2>
<p>Welcome to the repository of Bytecoin. Here you will find source code, instructions, wiki resources, and integration tutorials.</p>
<p>Contents</p>
<ul>
<li>Building on Linux 64-bit</li>
<li>Building on Mac OSX</li>
<li>Building on Windows</li>
<li>Building on other platforms</li>
</ul>
<h2><a id="user-content-building-on-linux-64-bit" class="anchor" aria-hidden="true" href="#building-on-linux-64-bit"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>Building on Linux 64-bit</h2>
<p>All commands below are adapted for Ubuntu, other distributions may need an other command set.</p>
<h3><a id="user-content-building-with-standard-options" class="anchor" aria-hidden="true" href="#building-with-standard-options"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>Building with standard options</h3>
<p>To go futher you have to have a number of packages and utilities.</p>
<ul>
<li>
<p><code>build-essential</code> package:</p>
<pre><code>$&gt; sudo apt-get install build-essential
</code></pre>
</li>
<li>
<p>CMake (3.5 or newer):</p>
<pre><code>$&gt; sudo apt-get install cmake
$&gt; cmake --version
</code></pre>
<p>If version is too old, follow instructions on <a href="https://cmake.org/download/" rel="nofollow">the official site</a>.</p>
</li>
<li>
<p>Boost (1.58 or newer):</p>
<pre><code>$&gt; sudo apt-get install libboost-all-dev
$&gt; cat /usr/include/boost/version.hpp | grep "BOOST_LIB_VERSION"
</code></pre>
<p>If version is too old, follow instructions on <a href="http://www.boost.org/users/download/" rel="nofollow">the official site</a>.</p>
</li>
</ul>
<p>Then create directory <code>bcndev</code> somewhere and go there:</p>
<pre><code>$&gt; mkdir bcndev
$&gt; cd bcndev
</code></pre>
<p>Git-clone (or git-pull) Bytecoin source code in that folder:</p>
<pre><code>$bcndev&gt; git clone https://github.com/bcndev/bytecoin.git
</code></pre>
<p>Put LMDB source code in <code>bcndev</code> folder (source files are referenced via relative paths, so you do not need to separately build it):</p>
<pre><code>$bcndev&gt; git clone https://github.com/LMDB/lmdb.git
</code></pre>
<p>Create build directory inside bytecoin, go there and run CMake and Make:</p>
<pre><code>$bcndev&gt; mkdir bytecoin/build
$bcndev&gt; cd bytecoin/build
$bcndev/bytecoin/build&gt; cmake ..
$bcndev/bytecoin/build&gt; time make -j4
</code></pre>
<p>Check built binaries by running them from <code>../bin</code> folder</p>
<pre><code>$bcndev/bytecoin/build&gt; ../bin/bytecoind -v
</code></pre>
<h3><a id="user-content-building-with-specific-options" class="anchor" aria-hidden="true" href="#building-with-specific-options"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>Building with specific options</h3>
<p>Install OpenSSL to <code>bcndev/openssl</code> folder. (Use switch <code>linux-x86_64-clang</code> instead of <code>linux-x86_64</code> if using clang.)</p>
<pre><code>$bcndev&gt; git clone https://github.com/openssl/openssl.git
$bcndev&gt; cd openssl
$bcndev/openssl&gt; ./Configure linux-x86_64
$bcndev/openssl&gt; time make -j4
$bcndev/openssl&gt; cd ..
</code></pre>
<p>Download amalgamated <a href="https://www.sqlite.org/download.html" rel="nofollow">SQLite 3</a> and unpack it into <code>bcndev/sqlite</code> folder (source files are referenced via relative paths, so you do not need to separately build it).</p>
<p>Below are the commands which add OpenSSL support and switch from LMDB to SQLite by providing options to CMake:</p>
<pre><code>$bcndev&gt; mkdir bytecoin/build
$bcndev&gt; cd bytecoin/build
$bcndev/bytecoin/build&gt; cmake -DBYTECOIN_SSL=1 -DBYTECOIN_SQLITE=1 ..
$bcndev/bytecoin/build&gt; time make -j4
</code></pre>
<h2><a id="user-content-building-on-mac-osx" class="anchor" aria-hidden="true" href="#building-on-mac-osx"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>Building on Mac OSX</h2>
<h3><a id="user-content-building-with-standard-options-1011-el-capitan-or-newer" class="anchor" aria-hidden="true" href="#building-with-standard-options-1011-el-capitan-or-newer"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>Building with standard options (10.11 El Capitan or newer)</h3>
<p>You need command-line tools. Either get XCode from an App Store or run 'xcode-select --install' in terminal and follow instructions. First of all, you need <a href="https://brew.sh" rel="nofollow">Homebrew</a>.</p>
<p>Then open terminal and install CMake and Boost:</p>
<ul>
<li><code>brew install cmake</code></li>
<li><code>brew install boost</code></li>
</ul>
<p>Create directory <code>bcndev</code> somewhere and go there:</p>
<pre><code>$~/Downloads&gt; mkdir &lt;path-to-bcndev-folder&gt;
$~/Downloads&gt; cd &lt;path-to-bcndev-folder&gt;
</code></pre>
<p>Git-clone (or git-pull) Bytecoin source code in that folder:</p>
<pre><code>$bcndev&gt; git clone https://github.com/bcndev/bytecoin.git
</code></pre>
<p>Put LMDB source code in <code>bcndev</code> folder (source files are referenced via relative paths, so you do not need to separately build it):</p>
<pre><code>$bcndev&gt; git clone https://github.com/LMDB/lmdb.git
</code></pre>
<p>Create build directory inside bytecoin, go there and run CMake and Make:</p>
<pre><code>$bcndev&gt; mkdir bytecoin/build
$bcndev&gt; cd bytecoin/build
$bcndev/bytecoin/build&gt; cmake ..
$bcndev/bytecoin/build&gt; time make -j4
</code></pre>
<p>Check built binaries by running them from <code>../bin</code> folder:</p>
<pre><code>$bcndev/bytecoin/build&gt; ../bin/bytecoind -v
</code></pre>
<h3><a id="user-content-building-with-specific-options-1" class="anchor" aria-hidden="true" href="#building-with-specific-options-1"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>Building with specific options</h3>
<p>Binaries linked with Boost installed by Homebrew will work only on your computer's OS X version or newer, but not on older versions like El Capitan.</p>
<p>If you need binaries to run on all versions of OS X starting from El Capitan, you need to build boost yourself targeting El Capitan SDK.</p>
<p>Download <a href="https://github.com/phracker/MacOSX-SDKs/releases">Mac OSX 10.11 SDK</a> and unpack to it into <code>Downloads</code> folder</p>
<p>Download and unpack <a href="https://boost.org" rel="nofollow">Boost</a> to <code>Downloads</code> folder.</p>
<p>Then build and install Boost:</p>
<pre><code>$~&gt; cd ~/Downloads/boost_1_58_0/
$~/Downloads/boost_1_58_0&gt; ./bootstrap.sh
$~/Downloads/boost_1_58_0&gt; ./b2 -a -j 4 cxxflags="-stdlib=libc++ -std=c++14 -mmacosx-version-min=10.11 -isysroot/Users/user/Downloads/MacOSX10.11.sdk" install`
</code></pre>
<p>Install OpenSSL to <code>bcndev/openssl</code> folder:</p>
<pre><code>$~/Downloads/bcndev&gt; git clone https://github.com/openssl/openssl.git
$~/Downloads/bcndev&gt; cd openssl
</code></pre>
<p>If you need binaries to run on all versions of OS X starting from El Capitan, you need to build OpenSSL targeting El Capitan SDK.</p>
<pre><code>$bcndev/openssl&gt; ./Configure darwin64-x86_64-cc no-shared -mmacosx-version-min=10.11 -isysroot/Users/user/Downloads/MacOSX10.11.sdk
</code></pre>
<p>Otherwise just use</p>
<pre><code>$bcndev/openssl&gt; ./Configure darwin64-x86_64-cc no-shared
</code></pre>
<pre><code>$bcndev/openssl&gt; time make -j4
$bcndev/openssl&gt; cd ..
</code></pre>
<p>Download amalgamated <a href="https://www.sqlite.org/download.html" rel="nofollow">SQLite 3</a> and unpack it into <code>bcndev/sqlite</code> folder (source files are referenced via relative paths, so you do not need to separately build it).</p>
<p>You add OpenSSL support or switch from LMDB to SQLite by providing options to CMake:</p>
<pre><code>$bcndev&gt; mkdir bytecoin/build
$bcndev&gt; cd bytecoin/build
$bcndev/bytecoin/build&gt; cmake -DBYTECOIN_SSL=1 -DBYTECOIN_SQLITE=1 ..
$bcndev/bytecoin/build&gt; time make -j4
</code></pre>
<h2><a id="user-content-building-on-windows" class="anchor" aria-hidden="true" href="#building-on-windows"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>Building on Windows</h2>
<p>You need Microsoft Visual Studio Community 2017. <a href="https://microsoft.com" rel="nofollow">Download</a> and install it selecting <code>C++</code>, <code>git</code>, <code>cmake integration</code> packages.</p>
<p>Get <a href="https://boost.org" rel="nofollow">Boost</a> and unpack it into a folder of your choice. We will use <code>C:\boost_1_58_0</code> in the further examples.</p>
<p>Run <code>Visual Studio x64 command prompt</code> from start menu.</p>
<p>Build boost</p>
<pre><code>$&gt; cd C:\boost_1_58_0
$C:\boost_1_58_0&gt; bootstrap.bat
$C:\boost_1_58_0&gt; b2.exe address-model=64 link=static
</code></pre>
<p>Set boost environment variables, right-click Computer in start menu, select <code>Properties</code>, then click <code>advanced system settings</code>, <code>environment variables</code>.</p>
<p>Set <code>BOOST_ROOT</code> to <code>C:\boost_1_58_0</code></p>
<p>Set <code>BOOST_INCLUDEDIR</code> to <code>C:\boost_1_58_0</code></p>
<p>Set <code>BOOST_LIBRARYDIR</code> to <code>C:\boost_1_58_0\stage\lib</code></p>
<p>Now create directory <code>bcndev</code> somewhere</p>
<pre><code>$C:\&gt; mkdir bcndev
$C:\&gt; cd bcndev
</code></pre>
<p>You need bytecoin source code</p>
<pre><code>$C:\bcndev&gt; git clone https://github.com/bcndev/bytecoin.git
</code></pre>
<p>You need lmdb in the same folder (source files are referenced via relative paths, so you do not need to separately build it)</p>
<pre><code>$C:\bcndev&gt; git clone https://github.com/LMDB/lmdb.git
</code></pre>
<p>Now launch Visual Studio, in File menu select <code>Open Folder</code>, select <code>C:\bcndev\bytecoin</code> folder.
Wait until CMake finishes running and <code>Build</code> appears in main menu.
Select <code>x64-Debug</code> or <code>x64-Release</code> from standard toolbar, and then <code>Build/Build Solution</code> from the main menu.</p>
<p>You cannot add options to CMake running inside Visual Studio so just edit <code>CMakeLists.txt</code> and set <code>BYTECOIN_SSL</code> or <code>BYTECOIN_SQLITE</code> to <code>ON</code> if you wish to build with them.</p>
<h2><a id="user-content-building-on-32-bit-x86-platforms-ios-android-and-other-arm-platforms" class="anchor" aria-hidden="true" href="#building-on-32-bit-x86-platforms-ios-android-and-other-arm-platforms"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>Building on 32-bit x86 platforms, iOS, Android and other ARM platforms</h2>
<p>Bytecoin works on 32-bit systems if SQLite is used instead of LMDB (we've experienced lots of problems building and running with lmdb in 32-bit compatibility mode, especially on iOS).</p>
<p>Therefore SQLite option is automatically selected by CMake on 32-bit platforms and you must have SQLite downloaded as explained in appropriate sections above.</p>
<p>We build official x86 32-bit version for Windows only, because there is zero demand for 32-bit version for Linux or Mac.</p>
<p>Building source code for iOS, Android, Raspberry PI, etc is possible (we have experimental <code>bytecoind</code> and <code>walletd</code> running on ARM64 iPhone) but requires major skills on your part. <strong>TBD</strong></p>
<h2><a id="user-content-building-on-big-endian-platforms" class="anchor" aria-hidden="true" href="#building-on-big-endian-platforms"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>Building on Big-Endian platforms</h2>
<p>Currently it is impossible to run Bytecoin on any Big-Endian platform, due to lots of endianess-dependent code. This may be fixed in the future. If you wish to run on Big-Endian platform, please contact us.</p>
<h2><a id="user-content-building-with-parameters" class="anchor" aria-hidden="true" href="#building-with-parameters"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>Building with parameters</h2>
<p>If you want to use tools like <code>clang-tidy</code>, run <code>cmake -DCMAKE_EXPORT_COMPILE_COMMANDS=ON ..</code> instead of <code>cmake ..</code></p>
